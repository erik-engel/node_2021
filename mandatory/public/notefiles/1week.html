First of all. What is node?


Simply put Node allows developers to write JavaScript code that will run directly in a computer process itself, and not in a browser. Therefore we can use node for writing server-side applications.

node -v or node --version

If the return msg is something like v15.8.0, node is already installed. If not node we will install node via the snap store


sudo snap install node --classic

Now I can check for the version once again, and the turnout should tell med which node version I’ve got installed at the moment Now that node is installed we can make a simple test.js file in vscode and run it with node. Basic Javascript


For a long time “Var” declarations was the way to go. But there are some issues associated with “Var”. These issues are concerning “Var”s scope. Var declarations are globally scoped or function scoped. Therefore if we do something like this:


var greet = “Hi there”
var x = 4 if (x > 3) {
     var greet = “Hi there, but different”
}
console.log(greet) // output will be “Hi there, but different”

as we can see the value will remain as the value declared withing the if-statement


"let" on the other hand is, as with "const", block-scoped. This means, that if you change “var” in the code from last example, to “let” the output will be “Hi there”. This is because it’s block scoped, and the code will treat the two “let greet” variables as being two different variables, one initialized within the blocks of the if statement and one initialized in the global scope.


Variables that are declared with “const” will, as the name almost states, remain constant. “const” can therefore not be updated, nor redeclared. Although if you declare a const object we can change it with following example:


const greeting = {
     msg: “Hello there”,
     x : 4
}
greeting.msg = “Hello there, but different”

“const” must be initialized during declaration. “var” and “let” don't have to be. Another difference between var and (let and const) is their behaviour when hoisted. Hoisting is JavaScript’s default behaviour of moving all declarations to the top of the current scope (to the top of the current script or the current function). When declared with “let” and “const” the variables are hoisted to the top of the block, but NOT initialized. With “var” the variable is hoisted to the top of the scope, and initialized with value of undefined.
